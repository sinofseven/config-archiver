# Kanban ワークフロー詳細手順書

## ログ記録の原則

ログはインタラクティブセッションで得た情報・判断・経緯の**完全な記録**である。kanban ファイルの `## プラン` は要約版、ログは完全版。**情報の要約・省略・圧縮をしてはならない。**

- 調べたファイルごとに発見した事実を具体的に書く（結論だけでなく内容そのものを記録する）
- プランモードでユーザーに提示した内容をそのまま記録する（圧縮しない）
- 会話・やり取りも省略せず時系列で記録する

## 命名規則

- ファイル名: `{xxxx}_{title}.md`
- `xxxx`: 4桁の0パディング連番（例: `0001`, `0002`）
- `title`: 作業タイトル（スペースなし、ハイフン区切り推奨）
- `kanban/` と `logs/` で同じファイル名を使用する

## タイムスタンプ

JSTタイムゾーンの ISO 8601 形式を使用する。

```bash
TZ=Asia/Tokyo date +"%Y-%m-%dT%H:%M:%S+09:00"
```

出力例: `2026-04-11T21:30:00+09:00`

## kanban ファイルへの追記テンプレート

kanban ファイルへの追記は以下の構造で行う。タスク内容はユーザーが記述し、以降の セクションを Claude が追記する。

```markdown
## プラン

（プランモードで作成した実装計画をここに記載）

- 実施内容の概要
- 変更するファイル・コンポーネント
- 採用したアプローチと理由

---

## 完了サマリー

- **完了日時**: 2026-04-11T21:30:00+09:00
- **対応内容**:
  - 実施した変更の箇条書き
- **変更ファイル**:
  - `path/to/file1.rs`
  - `path/to/file2.rs`
- **備考**: （任意）
```

## ログファイルテンプレート

`logs/{xxxx}_{title}.md` に以下の構造で記録する。

```markdown
# {xxxx}_{title} 作業ログ

## 基本情報

- **タスクファイル**: kanban/{xxxx}_{title}.md
- **開始日時**: 2026-04-11T21:00:00+09:00
- **完了日時**: 2026-04-11T21:30:00+09:00

## タスク概要

（kanban ファイルの要望セクションの内容を転記する）

## 調査結果

（フェーズ1で調査した内容を**省略せず詳細に**記述する。
調べたファイルごとに、発見した具体的な事実・構造・パターンを記述すること。
「〜を確認した」のような結論だけでなく、確認した内容そのものを書く。要約禁止。）

## 実装プラン

（インタラクティブセッションでの議論を元に、完全な実装プランを記述する。
kanban ファイルの `## プラン` は要約版であり、ログには完全版を書くこと。
検討した代替案・却下理由・採用アプローチとその理由、具体的なコードパスを含める。
プランモードでユーザーに提示した内容をそのまま記録すること（圧縮しない）。）

## プランニング経緯

### 初回提案

（最初に提示したプランの概要）

### ユーザーフィードバック

（リジェクト・修正要求の内容。リジェクトがなかった場合は「初回提案がそのまま承認された」と記載）

### 最終プラン

（初回から変更があった場合のみ記載。変更内容と採用理由を書く）

## 会話内容

（ユーザーの指示とClaudeの応答を時系列で記録）

### [21:00] ユーザー指示
（指示内容）

### [21:05] Claude 対応
（応答・実施内容）

## 編集したファイル

| ファイル | 変更内容 |
|---------|---------|
| `path/to/file.rs` | 変更の概要 |

## 実行したコマンド

```bash
cargo build
cargo test
```

## 判断・意思決定

- （設計判断やトレードオフの記録）

## エラー・問題

- （発生した問題と解決方法）
```

## 段階的ログ記録ルール

**重要**: ログは作業完了後にまとめて書くのではなく、段階的に追記すること。

1. **タスク開始時（最初のステップ）**: ログファイルを作成し、フェーズ1の成果を記入する
   - ヘッダー、基本情報（開始時刻）を書く（完了日時は TBD）
   - 「タスク概要」に kanban ファイルの要望を転記する
   - 「調査結果」にフェーズ1で調査した内容を**省略せず詳細に**記述する（調べたファイルごとに発見した事実を具体的に書く。要約禁止）
   - 「実装プラン」にインタラクティブセッションで議論した**完全な**実装プランを記述する（kanban ファイルの要約版ではなく、代替案・却下理由・採用理由を含む完全版）
   - 「プランニング経緯」にプランの変遷を記録する（初回提案・フィードバック・最終プラン）
   - 実装フェーズのセクション（編集したファイル、実行したコマンド、判断・意思決定、エラー・問題）は空テンプレートとして書く

2. **作業中（各ステップ完了時）**: ログファイルへ追記する
   - ファイルを編集したら「編集したファイル」セクションに追記
   - コマンドを実行したら「実行したコマンド」セクションに追記
   - 重要な判断をしたら「判断・意思決定」セクションに追記
   - エラーが発生したら「エラー・問題」セクションに追記

3. **作業完了時**: 最終化する
   - ログファイルの完了日時を更新する
   - 「会話内容」セクションにフェーズ2でのやり取りを追記する（省略せず記述する）
   - kanban ファイルへ完了サマリーを追記する

## タスク検出ロジック

- 未完了タスク: `kanban/` 内の `.md` ファイルで `## 完了サマリー` を含まないもの
- `/kanban` コマンドに引数がない場合: 未完了タスクのうち番号が最大のものを選択
