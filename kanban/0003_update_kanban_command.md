# `/kanban` コマンドのログ出力内容を更新
## 要望

今に至っても、ログの内容が薄い。

Claude Codeのインタラクティブセッションで表示されるプランの内容と比べると薄い。
ログは記録用の場所だから、要約せずにログに残して欲しい。

## プラン

**根本原因**: コマンド定義の「調査結果」指示が「まとめる」という要約を促す動詞を使っており、「要約禁止」の明確な原則もないため、Claude が薄いログを書いてしまう。

**対応内容**:
1. `.claude/commands/kanban.md` — フェーズ2 冒頭に「ログ＝完全記録、要約禁止」の原則を追加。「調査結果」の「まとめる」を「省略せず詳細に記述する」に変更。「実装プラン」に「圧縮しない」を明記。「会話内容」の初期記入指示を追加。
2. `.claude/kanban-workflow.md` — 冒頭に「ログ記録の原則」セクションを追加。テンプレート・段階的記録ルールの要約を促す表現をすべて排除。
