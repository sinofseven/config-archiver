# `/kanban`コマンドのログ出力を更新する
## 要望
`/kanban` コマンドで生成するログ出力において、プランの内容がインタラクティブセッションで表示されたものと比べると内容が浅い。
インタラクティブセッションで表示されたプランの内容をログ出力にも書き込んで欲しい。

また、プランに対してリジェクトしたり追加の質問で内容を変えたときは、それも記録して欲しい。

## プラン

ログ出力が浅くなる原因は3つ: (1) kanban ファイルの簡潔なプランをそのまま転記しているため情報が2段階で圧縮される、(2) プランのリジェクト・修正の経緯を記録する場所がない、(3) `## 会話内容` セクションが完了後まとめる指示のため実質スキップされる。

### 変更対象ファイル
- `.claude/commands/kanban.md` — コマンド定義（Claude への直接的な指示）
- `.claude/kanban-workflow.md` — ワークフロー詳細手順書（テンプレート・ルール）

### 変更方針

**`.claude/commands/kanban.md`**
- フェーズ1: プランリジェクト時の記録手順を追加（リジェクト内容・理由を保持、最終プランで kanban ファイルを更新）
- フェーズ2 ステップ1: ログの「実装プラン」を「転記」から「インタラクティブセッションの詳細内容を元に完全版を記述」に変更（代替案・却下理由・採用理由を含める）
- フェーズ2 ステップ1: 新セクション「プランニング経緯」の記入指示を追加（初回提案・フィードバック・最終プラン）

**`.claude/kanban-workflow.md`**
- ログテンプレートの `## 実装プラン` 説明を「転記」から「詳細記述」に変更
- `## 実装プラン` と `## 会話内容` の間に `## プランニング経緯` セクションを追加
- 段階的ログ記録ルール（ステップ1）に「プランニング経緯」セクションの記入を追加
