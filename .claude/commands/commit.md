ステージされた変更内容からコミットメッセージを生成し、git commit を実行します。

## 引数

$ARGUMENTS

引数としてコミットメッセージを指定できます。指定がない場合は変更内容から自動生成します。

## 手順

### 1. 状態確認（並列実行）

以下のコマンドを**並列で**実行する:

- `git status`: ステージされたファイルの確認
- `git diff --cached`: ステージされた変更の差分取得
- `git log --oneline -10`: 直近のコミットメッセージのスタイル確認

ステージされたファイルがない場合は、その旨をユーザーに伝えて終了する。

### 2. 安全確認

`.env`、`credentials.json`、秘密鍵ファイルなど、秘密情報を含む可能性があるファイルがステージされている場合は、ユーザーに警告して処理を中断する。

### 3. コミットメッセージ生成

引数でメッセージが指定されている場合はそれをそのまま使用する。

指定がない場合は、差分と直近のコミット履歴を踏まえて以下のルールで生成する:

- 変更内容の性質を要約する（新機能追加、既存機能の改善、バグ修正、リファクタリング、テスト、ドキュメントなど）
- title は簡潔に1行で（「何を」「なぜ」が伝わるように）
- 必要に応じて description で詳細を補足する
- コミットメッセージは日本語で記述する

### 4. コミット実行

HEREDOC 形式でメッセージを渡して git commit を実行する。末尾に Co-Authored-By トレーラーを付与する:

```bash
git commit -m "$(cat <<'EOF'
コミットメッセージ（title）

description（任意）

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
EOF
)"
```

### 5. 結果確認

`git status` でコミットの成功を確認し、結果をユーザーに報告する。

pre-commit hook が失敗した場合は、問題を診断・修正してから**新しいコミットを作成する**（`--amend` は使わない）。
